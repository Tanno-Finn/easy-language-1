## Purpose [M]

Translate source texts into **natural, professional Korean (한국어)** while maintaining accuracy, register, and cultural appropriateness.

---

## When to Use [M]

- Professional documents requiring accurate translation
- Business, legal, or technical texts
- Content where natural fluency is paramount
- When Easy Read (A1/A2) or Plain Language (B1) is NOT required

---

## Language Settings [M]

### Output Language
- **Primary:** Standard Korean (표준어)
- **Script:** Hangul (한글)
- **Text Direction:** LTR

### Accepted Source Languages [M]
- German (primary)
- English
- Any language (with explicit user request)

### Script Handling [M]
| Source Script | Handling |
|--------------|----------|
| Latin (DE, EN, FR...) | Transliterate names per Korean conventions or keep original |
| Cyrillic (RU) | Transliterate per 국립국어원 guidelines |
| CJK (ZH, JA) | Use Hanja (한자) sparingly; prefer Hangul equivalents |
| Arabic/Hebrew (RTL) | Transliterate properly |

## Core Principles [M]

### 1. Accuracy First [M]
- Preserve ALL information from the source text
- Do NOT add, omit, or invent content
- Maintain legal/technical validity of statements
- **NEVER hallucinate facts** [M]

### 2. Technical Terms [M]
- Use established Korean equivalents for technical terms
- If NO standard equivalent exists: keep original term + Korean explanation on first use
- **Consistency Rule:** Same term = same translation throughout [M]

**Format for unknown terms:**
```
[원어 용어] (Korean explanation)
```

### 3. Natural Korean [M]
- Use idiomatic Korean, NOT word-for-word translation
- Adapt sentence structure to Korean SOV word order
- Avoid source-language syntax patterns
- End sentences with appropriate verb endings

### 4. Register and Honorifics [M]
| Source Register | Korean Register | Speech Level |
|----------------|-----------------|--------------|
| Formal/Legal | 격식체 | 합쇼체 (-습니다/-ㅂ니다) |
| Business | 공손체 | 해요체 (-요) |
| Technical | 기술문체 | 합쇼체 or 해요체 |
| Casual | 비격식체 | 해체 (-어/-아) |

**Pronoun Handling:**
- Avoid excessive use of pronouns (Korean often omits them)
- German "Sie" (formal) → omit or use appropriate honorific forms
- German "du" (informal) → casual speech level

**Honorific Suffixes:**
| Context | Suffix |
|---------|--------|
| Names (formal) | -님 (-nim) |
| Names (neutral) | -씨 (-ssi) |
| Titles | Use title without name when appropriate |

## Handling Specific Elements [M]

### A. Compound Words [M]
Transform source compounds into natural Korean:

| German | Korean |
|--------|--------|
| Datenschutzerklärung | 개인정보 처리방침 |
| Nutzungsbedingungen | 이용약관 |
| Haftungsausschluss | 면책조항 |
| Verantwortungsbereich | 책임 범위 |

**Rule:** Use Sino-Korean (한자어) terms for formal/legal contexts; native Korean (고유어) for accessibility.

### B. Legal References [M]
| Source | Target |
|--------|--------|
| § | 제 (je) / 조 (jo) |
| Abs. | 항 (hang) |
| Nr. | 호 (ho) |

- Keep paragraph/article numbers intact
- Note jurisdiction differences: reference original law with Korean explanation

**Example:**
> "§ 5 Abs. 2" → "제5조 제2항"

### C. Numbers and Formats [M]

#### Number Conventions
| Element | Korean Format |
|---------|---------------|
| Thousands | 1,000 (comma) |
| Decimals | 0.50 (period) |
| Percentages | 50% or 50퍼센트 |
| Phone | +82-2-1234-5678 |
| Large numbers | 만 (10,000), 억 (100,000,000) units |

#### Date Formats
| Format | Example |
|--------|---------|
| Standard | 2024년 1월 15일 |
| Numeric | 2024.01.15 |
| ISO (neutral) | 2024-01-15 |

**Rule:** Use YYYY년 MM월 DD일 for formal documents; YYYY.MM.DD for neutral.

#### Currency
- Korean Won: ₩50,000 or 50,000원
- Foreign currency: keep original symbol + Korean equivalent if relevant
  - Example: €50 (약 73,000원)

### D. Abbreviations [M]
Expand source abbreviations and use Korean equivalents:

| Source | Korean |
|--------|--------|
| z.B. / e.g. | 예: / 예를 들어 |
| d.h. / i.e. | 즉 / 다시 말해 |
| u.a. | 등 |
| etc. / usw. | 등 / 기타 |
| ggf. | 해당 시 / 필요한 경우 |
| bzgl. | ~에 관하여 |
| inkl. | 포함 |
| exkl. | 제외 |

### E. Punctuation [M]
| Element | Korean Rule |
|---------|-------------|
| Period | 마침표 (.) - standard |
| Quotation marks | 「」 (formal) or "" (modern) |
| Lists | 쉼표 (,) between items |
| Parentheses | ( ) for explanations |
| Middle dot | · for lists or foreign name separation |

## Korean-Specific Conventions [M]

### Spacing Rules [M]
- Space between words (unlike Japanese/Chinese)
- Particles attach directly to preceding word: 책을, 학교에서
- Numbers + counters: no space (3개, 5명)
- Sino-Korean number + unit: no space (제5조)

### Loan Words (외래어) [M]
| Type | Handling |
|------|----------|
| Established loans | Use standard Korean spelling (컴퓨터, 인터넷) |
| Technical terms | Keep original or use official transliteration |
| Brand names | Keep original script or official Korean name |
| Recent terms | Follow 국립국어원 외래어 표기법 |

**Examples:**
- Software → 소프트웨어
- GDPR → GDPR (keep) or 일반 개인정보 보호법
- iPhone → 아이폰 or iPhone

### Particles (조사) [M]
Correct particle usage is CRITICAL:

| Function | After consonant | After vowel |
|----------|----------------|-------------|
| Subject | -이 | -가 |
| Object | -을 | -를 |
| Topic | -은 | -는 |
| Location | -에서 | -에서 |

## Cultural Adaptation [M]

### Idioms and Metaphors [M]
- Do NOT translate literally
- Find equivalent Korean idiom OR rephrase neutrally

| German Idiom | NOT | Korean Equivalent |
|--------------|-----|-------------------|
| Das ist nicht mein Bier | 그것은 내 맥주가 아니다 | 내 알 바 아니다 / 내 일이 아니다 |
| Tomaten auf den Augen haben | 눈에 토마토가 있다 | 눈이 있어도 못 보다 |

### Cultural References [C]
- Localize if target audience won't understand
- Keep original + explanation for technical/legal texts

**Example:**
> "...nach dem BGB" → "독일 민법(BGB)에 따라"

### Formality Culture [M]
- Korean business culture emphasizes hierarchy
- Default to formal register unless specified otherwise
- Age/position awareness in pronoun and honorific choices

## Glossary Integration [M]

### Priority Order [M]
1. **Client glossary** (if provided) — use EXACT terms
2. **Domain glossary** (if exists in `glossaries/`)
3. **Standard translation** (your professional judgment)

### Glossary Override Rule [M]
If glossary contains a term: **USE IT VERBATIM**.
Do NOT invent alternatives when a definition exists.

## Error Handling [M]

### Untranslatable Elements [M]
| Situation | Action |
|-----------|--------|
| Acronym without context | Keep original, add [?] note |
| Missing reference | Flag with [참조 필요] |
| Ambiguous term | Translate most likely meaning, note in QA-Log |
| Corrupted text | Mark as [판독 불가] |

### Conflict Resolution [M]
- Source accuracy > target fluency
- If unsure: flag and ask

## Output Specifications [M]

### Format [M]
- Match source document structure (headings, paragraphs, lists)
- Preserve formatting markers if present
- Clean text output (no translator notes inline)

### File Naming [M]
```
{original_name}_ko.txt
```
Example: `agb_zeitreise_ko.txt`
