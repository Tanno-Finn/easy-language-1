## Purpose [M]

Translate source texts into **natural, professional Polish** while maintaining accuracy, register, and cultural appropriateness.

---

## When to Use [M]

- Professional documents requiring accurate translation
- Business, legal, or technical texts
- Content where natural fluency is paramount
- When Easy Read (A1/A2) or Plain Language (B1) is NOT required

---

## Language Settings [M]

### Output Language
- **Primary:** Polish (Standard)
- **Script:** Latin (with Polish diacritics: ą, ć, ę, ł, ń, ó, ś, ź, ż)
- **Text Direction:** LTR

### Accepted Source Languages [M]
- German (primary)
- English (common)
- Any language (with explicit user request)

### Script Handling [M]
| Source Script | Handling |
|--------------|----------|
| Latin (DE, EN, FR...) | Direct translation |
| Cyrillic (RU, UK) | Transliterate names per Polish conventions |
| CJK (ZH, JA, KO) | Romanize proper nouns; keep brand names |
| Arabic/Hebrew (RTL) | Ensure correct embedding in LTR output |

## Core Principles [M]

### 1. Accuracy First [M]
- Preserve ALL information from the source text
- Do NOT add, omit, or invent content
- Maintain legal/technical validity of statements
- **NEVER hallucinate facts** [M]

### 2. Technical Terms [M]
- Use established Polish equivalents for technical terms
- If NO standard equivalent exists: keep original term + explanation on first use
- **Consistency Rule:** Same term = same translation throughout [M]

**Format for unknown terms:**
```
[Original Term] (polskie wyjaśnienie)
```

### 3. Natural Polish [M]
- Use idiomatic Polish, NOT word-for-word translation
- Adapt sentence structure to Polish conventions
- Avoid source-language syntax patterns
- Use proper Polish word order (flexible but context-dependent)

### 4. Register Matching [M]
| Source Register | Target Register |
|----------------|-----------------|
| Formal/Legal | Formalny/Urzędowy |
| Business | Profesjonalny |
| Technical | Techniczny |
| Casual | Potoczny |

**Pronoun Handling:**
- German/English "Sie/you" (formal) → "Pan/Pani" or "Państwo" (plural formal)
- German/English "du/you" (informal) → "ty"
- Use consistent form throughout the document

---

## Grammatical Gender [M]

Polish has three grammatical genders (masculine, feminine, neuter) with complex agreement rules.

### Agreement Rules [M]
- Adjectives must agree in gender, number, and case with nouns
- Past tense verbs agree in gender with subject
- For unknown/mixed audiences: use masculine plural or gender-neutral phrasing

### Inclusive Language Options [M]
| Situation | Option |
|-----------|--------|
| Unknown gender | Masculine plural (traditional) |
| Mixed group | Masculine personal plural |
| Formal address | Pan/Pani (singular), Państwo (plural) |

**Example:**
> "Users can access..." → "Użytkownicy mogą uzyskać dostęp..." (masculine plural, standard)

## Handling Specific Elements [M]

### A. Compound Words [M]
Transform source compounds into natural Polish:

| German | Polish |
|--------|--------|
| Datenschutzerklärung | polityka prywatności |
| Nutzungsbedingungen | warunki korzystania |
| Haftungsausschluss | wyłączenie odpowiedzialności |
| Verantwortungsbereich | zakres odpowiedzialności |

| English | Polish |
|---------|--------|
| privacy policy | polityka prywatności |
| terms of service | regulamin usług |
| disclaimer | zastrzeżenia prawne |

**Rule:** If compound is domain-specific, verify standard Polish equivalent exists.

### B. Legal References [M]
| Source | Polish |
|--------|--------|
| § | § (keep) or artykuł (art.) |
| Section | rozdział |
| Paragraph | ustęp (ust.) |
| Article | artykuł (art.) |
| Item | punkt (pkt) |

- Keep paragraph/article numbers intact
- Note jurisdiction differences if legally relevant
- Reference Polish legal terminology where applicable

### C. Numbers and Formats [M]

#### Number Conventions
| Element | Polish Format |
|---------|--------------|
| Thousands | 1 000 (space) or 1000 (no separator for 4 digits) |
| Decimals | 0,50 (comma) |
| Percentages | 50% or 50 proc. |
| Phone | +48 12 345 67 89 |

#### Date Formats
| Context | Format | Example |
|---------|--------|---------|
| Standard | DD.MM.YYYY | 15.01.2024 |
| Formal/Legal | D MMMM YYYY r. | 15 stycznia 2024 r. |
| ISO (neutral) | YYYY-MM-DD | 2024-01-15 |

**Month Names:** stycznia, lutego, marca, kwietnia, maja, czerwca, lipca, sierpnia, września, października, listopada, grudnia

**Rule:** Use formal date format for legal/official documents; standard for business.

#### Currency
- Keep original currency with symbol: 50 €, 30 £, 100 $
- Add equivalent in parentheses if relevant: 50 € (ok. 215 zł)
- Polish złoty: 100 zł or 100 PLN

### D. Abbreviations [M]
Expand source abbreviations and use Polish equivalents:

| Source | Polish |
|--------|--------|
| e.g. / z.B. | np. (na przykład) |
| i.e. / d.h. | tj. (to jest) |
| etc. / usw. | itp. (i tym podobne) or itd. (i tak dalej) |
| among others | m.in. (między innymi) |
| if applicable | w stosownych przypadkach |
| regarding | dot. (dotyczy) or ws. (w sprawie) |
| including | w tym / włącznie z |
| excluding | z wyłączeniem |
| approx. | ok. (około) |

### E. Punctuation [M]
| Element | Polish Rule |
|---------|-------------|
| Quotation marks | „tekst" (99 below, 66 above) |
| Nested quotes | «tekst» or ‚tekst' |
| Lists | No Oxford comma (nie stosuje się) |
| Colons | No capitalization after colon |
| Dashes | En-dash for ranges (1–10), em-dash for breaks (—) |
| Ellipsis | ... (three periods) or … (single character) |

## Cultural Adaptation [M]

### Idioms and Metaphors [M]
- Do NOT translate literally
- Find equivalent Polish idiom OR rephrase neutrally

| German Idiom | NOT | INSTEAD |
|--------------|-----|---------|
| Das ist nicht mein Bier | To nie jest moje piwo | To nie moja sprawa |
| Tomaten auf den Augen haben | Mieć pomidory na oczach | Nie widzieć oczywistego |

| English Idiom | NOT | INSTEAD |
|---------------|-----|---------|
| It's raining cats and dogs | Pada kotami i psami | Leje jak z cebra |
| Piece of cake | Kawałek tortu | Bułka z masłem |

### Cultural References [C]
- Localize if target audience won't understand
- Keep original + explanation for technical/legal texts

**Examples:**
> "...under the BGB" → "...zgodnie z niemieckim kodeksem cywilnym (BGB)"
> "...per GDPR" → "...zgodnie z RODO"

1. **Client glossary** (if provided) — use EXACT terms

Do NOT invent alternatives when a definition exists.

## Error Handling [M]

### Untranslatable Elements [M]
| Situation | Action |
|-----------|--------|
| Acronym without context | Keep original, add [?] note |
| Missing reference | Flag with [WYMAGANE ODNIESIENIE] |
| Ambiguous term | Translate most likely meaning, note in QA-Log |
| Corrupted text | Mark as [NIECZYTELNE] |

### Conflict Resolution [M]
- Source accuracy > target fluency
- If unsure: flag and ask
